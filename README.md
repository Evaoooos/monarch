# 列城之主

五封战书压在案上，三千残兵等你一句命令。

父亲死在没有月亮的夜里。你继承的不是王位，而是一座破城、一座快空的粮仓，和一群还不知道该不该相信你的士兵。

从今天起，每一次挥手都是军令，每一次心软都会留下代价。你可以做仁君，也可以做暴君；可以攻下五城，一统天下；也可能死在饥荒、兵变、谣言、温柔乡，或某个最信任的人手里。

**《列城之主》是一款古风策略卡牌决策游戏：用一张张卡牌经营军政、权谋、人心与欲望，最终成为天下之主，或醉倒在温柔乡中。**

## 这是什么

这是一个策略卡牌类的网页互动游戏。
玩家扮演破城少主，在一张张左右抉择的事件卡中经营军政、粮草、人心与权谋，向五座城池步步推进。

乱世不会给你完美答案。每一次心软、强征、猜忌或沉溺，都会把后续事件推向不同方向；阿蛮、红袖、清越、巫月也会陆续入局，救你、误你、背叛你，或在某条结局里成为你最后放不下的人。

## 一句话描述

**《列城之主》是一款古风策略卡牌游戏：滑动卡牌做出决定，平衡军政、权谋、人心与欲望，是在乱世中攻下五城，成为天下之主，亦或醉倒在温柔乡中，把江山亲手葬送。记住，死亡不是终点，有些真相只会在轮回后出现。**

打开网址即玩，无需下载、无需安装，排队、摸鱼、午休随时随地来一局~：
```text
https://evaoooos.github.io/monarch/列城之主.html
```

## 主要看点

- **左右抉择** 左右滑动屏幕做出抉择，经营兵力、粮草、民心、谋略
- **红颜入局** 阿蛮、红袖、清越、巫月会陆续入局，改变后续事件、攻城策略和部分特殊结局
- **连环伏笔** 一次放人、一纸契约，几十个月后变成锁死命运的后续事件
- **死亡不是终点** 而是获得新的兵法，有些真相只会在轮回后出现。
- **30 张命运卡面** 每个结局配专属插画
- **摸鱼神器** 单手竖屏左右滑，秒开秒退，30 秒自动存档。
- **内容丰富** 300 余张事件卡牌，涵盖兵权决策与，拒绝机械重复；
              3W字白描文本，30 个多重结局；
              单局10分钟快节奏，全图鉴收集需 5-10 小时


## 用了什么实现

- HTML / CSS / JavaScript
- `localStorage` 本地存档
- Unity2.ai API，用于创作阶段生成结局插画
- 图片模型：`gpt-image-2`

核心文件：

- `列城之主.html`：游戏入口
- `game.css`：界面样式
- `engine.js`：卡牌数据、抽卡、数值、结局、存档、UI 逻辑
- `generate_ending_images.mjs`：结局插画批量生成脚本，开发时使用，玩家游玩不需要
- `images/ending_image_prompts.jsonl`：30 个结局插画 prompt 记录

## AI 使用说明

本项目是一次完整的 AI 辅助单人创作实验。

项目整体都深度使用了 AI 协作，包括：

- 核心玩法设计讨论
- 卡牌文案创作与扩展
- 结局文案与结局插画 prompt 设计
- JavaScript 游戏逻辑编写与调试
- HTML / CSS 界面实现与调整
- Unity2.ai API 批量生成结局插画
- 检查卡牌逻辑、结局触发和存档流程

人工完成和把关的部分包括：

- 确定最终游戏方向、选择保留哪些玩法、砍掉哪些机制
- 决定视觉风格、UI 取舍和移动端体验
- 调整数值难度和卡池节奏
- 审核生成图片是否适合游戏气质
