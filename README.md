# 列城之主

五封战书压在案上，三千残兵等你一句命令。

父亲死在没有月亮的夜里。你继承的不是王位，而是一座破城、一座快空的粮仓，和一群还不知道该不该相信你的士兵。

从今天起，每一次挥手都是军令，每一次心软都会留下代价。你可以做仁君，也可以做暴君；可以攻下五城，一统天下；也可能死在饥荒、兵变、谣言、温柔乡，或某个最信任的人手里。

**《列城之主》是一款古风乱世卡牌决策游戏：用一张张二选一卡牌经营军政、权谋、人心与欲望，最终成为天下之主，或死在自己的选择里。**

## 这是什么

这是一个参加 Unity2 创作比赛 C 组「游戏与互动体验」的网页互动作品。

玩法上是轻量的 Reigns 式左右抉择：玩家不断抽到事件卡，在两个选项中做决定，管理四项核心数值：

- 兵力
- 粮草
- 民心
- 谋略

目标是攻克五座城池。但四项数值都不是越高越好：太低会崩盘，太高也会反噬。军队太弱会兵变，兵权太盛会被将领架空；民心太低会暴动，民心太高又会把你推成危险的“圣人”；谋略不足会中伏，谋略过盛也会众叛亲离。

乱世路上，你会遇见四位关键女子：阿蛮、红袖、清越、巫月。她们可能救你、误你、背叛你，也可能在某条结局里成为你最后放不下的人。她们的出现会改变后续事件、攻城选择和命运走向。

## 一句话描述

**在乱世中靠二选一卡牌攻下五城，经营军政、权谋与人心，最终成为天下之主，或被自己的选择吞没。**

## 怎么玩

这是一个为手机摸鱼场景设计的网页小游戏：不用下载，不用安装，不用注册，也不需要任何 API key。

玩家直接打开 GitHub Pages 地址就能玩：

```text
提交时填写你的 GitHub Pages 链接
```

适合通勤、午休、排队、等人时随手开一局。游戏进度、结局图鉴和周目记录会保存在浏览器本地，关掉页面后下次再打开也能继续收集。

本地试玩也很简单：直接用浏览器打开 `列城之主.html`。

## 主要看点

- 265 张卡牌：逃兵、密报、粮仓、流民、刺客、内乱、攻城，每一张都逼你在代价之间下注
- 支持手机屏幕左右滑动选项，适合通勤、午休、摸鱼时随手开一局
- 五座城层层推进，每攻下一城，局势都会变得更危险
- 30 个结局，包括亡国、称王、失控、羁绊和隐藏结局
- 阿蛮、红袖、清越、巫月会逐步走入你的乱世，她们不是背景人物，而会改变你的牌局和结局
- 事件存在连锁选项：今天放走的人，可能会在下一座城前重新出现
- 支持多周目轮回，失败不会清空一切，有些线索只能在下一次出征中看见真相
- 四项国势不是越高越好：兵太盛会反噬，谋太深会众叛，民心太满也可能把你推上祭坛
- 结局页支持竖向结局插画，强化每局落幕时的仪式感

## 用了什么实现

- HTML / CSS / JavaScript
- `localStorage` 本地存档
- Unity2.ai API，用于创作阶段生成结局插画
- 图片模型：`gpt-image-2`

核心文件：

- `列城之主.html`：游戏入口
- `game.css`：界面样式
- `engine.js`：卡牌数据、抽卡、数值、结局、存档、UI 逻辑
- `generate_ending_images.mjs`：结局插画批量生成脚本，开发时使用，玩家游玩不需要
- `generate_card_images.mjs` / `generate_card_images.py`：早期卡牌图批量生图脚本，保留为开发资料
- `images/ending_image_prompts.jsonl`：30 个结局插画 prompt 记录
- `列城之主_游戏设计文档.md`：游戏设计文档

## AI 使用说明

本项目是一次完整的 AI 辅助单人创作实验。

项目整体都深度使用了 AI 协作，包括：

- 游戏主题方向讨论
- 核心玩法设计讨论
- 数值系统、攻城系统、多周目系统讨论
- 卡牌文案创作与扩展
- 结局文案与结局插画 prompt 设计
- JavaScript 游戏逻辑编写与调试
- HTML / CSS 界面实现与调整
- README 和设计文档整理
- Unity2.ai API 批量生成结局插画
- 生图脚本编写与参数调整

人工完成和把关的部分包括：

- 确定最终游戏方向与参赛定位
- 选择保留哪些玩法、砍掉哪些机制
- 决定视觉风格、UI 取舍和移动端体验
- 调整数值难度和卡池节奏
- 检查卡牌逻辑、结局触发和存档流程
- 审核生成图片是否适合游戏气质
- 整理最终项目结构并准备提交

也就是说，这不是只在最后“生成几张图”的 AI 使用，而是从创意、玩法、代码、文案、视觉到调试都深度使用 AI 协作完成的作品。

## 演示建议

演示时可以从这几个点切入：

1. 开局继承破城，展示第一张事件卡和四维数值。
2. 连续做几次选择，让观众看到数值不是越高越好。
3. 展示攻城卡，说明五座城是阶段目标。
4. 攻城后展示阿蛮、红袖、清越、巫月等人物登场，说明角色关系会改变后续事件。
5. 展示失败后获得「兵法」，说明多周目不是简单重开，有些真相要轮回后才能看见。
6. 打开结局图鉴，展示 30 个结局。
7. 展示结局页插画，说明视觉资源由 Unity2.ai API 辅助生成。
