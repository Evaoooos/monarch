# 列城之主

五封战书压在案上，三千残兵等你一句命令。

父亲死在一个没有月亮的夜里，旧朝崩塌，诸侯并起。你继承的不是王位，而是一座破城、一座快空的粮仓，和一群还不知道该不该相信你的士兵。

从今天起，每一次挥手都是军令，每一次迟疑都会死人。你可以做仁君，也可以做暴君；可以攻下五城，一统天下；也可能死在饥荒、兵变、谣言、温柔乡，或某个最信任的人手里。

**《列城之主》是一款古风乱世卡牌决策游戏：用二选一决定一座城的命运，也决定你会成为怎样的王。**

## 这是什么

这是一个参加 Unity2 创作比赛 C 组「游戏与互动体验」的网页互动作品。

玩法上参考了 Reigns 式左右抉择：玩家在一张张事件卡中做选择，管理四项核心数值：

- 兵力
- 粮草
- 民心
- 谋略

你的目标是攻克五座城池。但每项数值都不是越高越好：太低会崩盘，太高也可能反噬。军队太弱会兵变，兵权太盛会被将领架空；民心太低会暴动，民心太高又会把你推成危险的“圣人”。

游戏中还会逐步解锁四位关键 NPC：阿蛮、红袖、清照、巫月。
她们不是单纯的奖励角色，而会改变卡池、攻城节奏、好感关系和部分结局走向。

## 一句话描述

**在乱世中靠一张张二选一卡牌攻下五城，经营军政、权谋与人心，最终成为天下之主，或死在自己的选择里。**

## 怎么玩

这是一个给手机摸鱼场景准备的网页小游戏：不用下载，不用安装，不用注册，也不需要任何 API key。

## 主要看点

- 198 张事件卡，覆盖军政、民生、权谋、攻城和 NPC 关系
- 5 座攻城里程碑，每座城都有不同的数值要求
- 30 个结局，包括覆灭、羁绊、歧途和隐藏结局
- 四维数值都有低谷和满溢结局，不是简单堆高数值
- 四位 NPC 会逐步加入卡池，影响后续事件和攻城策略
- 多周目成长系统「兵法」，失败也会留下下一次出征的经验
- 移动端可滑动选卡，PC 端可点击或用键盘选择
- 卡牌插画由 Unity2.ai API 批量生成，统一成古风水墨卷轴风格

## 用了什么实现

- HTML / CSS / JavaScript
- `localStorage` 本地存档
- 创作阶段使用 Unity2.ai API 生成卡牌插画
- 图片模型：`gpt-image-2`

核心文件：

- `列城之主.html`：游戏入口
- `game.css`：界面样式
- `engine.js`：游戏数据、抽卡、结算、存档、UI 逻辑
- `images/`：卡牌插画
- `generate_card_images.mjs`：Node.js 批量生图脚本，开发时使用，玩家游玩不需要
- `generate_card_images.py`：Python 批量生图脚本，开发时使用，玩家游玩不需要
- `列城之主_游戏设计文档.md`：游戏设计文档

## AI 使用说明

本项目是一次完整的 AI 辅助单人创作实验。

使用 AI 辅助完成的部分包括：

- 游戏主题方向讨论
- 核心玩法设计讨论
- 数值系统和结局系统讨论
- 卡牌文案创作与扩展
- JavaScript 游戏逻辑编写与调试
- HTML / CSS 界面实现与调整
- 设计文档整理
- README 编写
- Unity2.ai API 批量生成卡牌插画
- 生图 prompt 设计与批量脚本编写

人工完成和把关的部分包括：

- 确定最终游戏方向与参赛定位
- 选择保留哪些玩法、砍掉哪些机制
- 调整数值难度和体验节奏
- 检查卡牌逻辑、结局触发和存档流程
- 审核生成图片是否适合游戏气质
- 整理最终项目结构并准备提交

也就是说，这不是只在最后“生成几张图”的 AI 使用，而是从创意、玩法、代码、文案、视觉到调试都深度使用 AI 协作完成的作品。


## 演示建议

演示时可以从这几个点切入：

1. 开局继承破城，展示第一张事件卡和四维数值。
2. 连续做几次选择，让观众看到数值并非单纯越高越好。
3. 展示攻城卡，说明五座城是游戏的阶段目标。
4. 展示攻城后 NPC 加入卡池，说明角色关系会改变后续事件。
5. 打开结局图鉴，展示多结局和多周目兵法系统。
6. 展示卡牌插画，说明视觉资源由 Unity2.ai API 批量生成。
