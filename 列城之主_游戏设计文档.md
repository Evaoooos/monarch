# 《列城之主》完整游戏设计文档

> 技术栈：单文件 HTML + CSS + JavaScript，纯前端，无后端依赖
> 参考原型：《Reigns》+ 姊妹作《王女背后的男人们》
> 目标平台：移动端 H5 / PC 浏览器

---

## 一、游戏概述

### 1.1 一句话描述
乱世崛起的年轻诸侯，通过二选一卡牌决策管理军政、攻克五座城池、收纳美人入后宫，最终统一天下——或死于途中。

### 1.2 核心循环
```
抽卡 → 读叙事 → 二选一 → 数值变化 → 检查结局/攻城 → 抽下一张卡
```

### 1.3 和《王女背后的男人们》的关键差异

| 维度 | 王女 | 列城之主 |
|------|------|---------|
| 目标 | 防守（活过五年） | 进攻（攻下五城） |
| NPC来源 | 开局全在 | 攻城解锁，逐步加入卡池 |
| NPC性别 | 4男 → 女主 | 4女 → 男主 |
| 里程碑 | 无（纯生存） | 5座城池检查点 |
| 隐藏属性 | 好感+残忍+仁慈 | 好感+杀伐+仁义+色欲 |
| 局外成长 | 王印（遗物） | 兵法（同等机制，换名换效果） |

## 二、世界观设定

### 2.1 背景
架空乱世。旧朝覆灭，天下分裂为数十个大小诸侯国。玩家扮演一个刚刚继承父亲领地的年轻诸侯。父亲死于一场不明不白的"意外"，留给你的只有一座破城、三千残兵，以及五封来自五座城池的战书。

### 2.2 五座城池
城池是本游戏的里程碑系统。每座城在玩家达到特定月份后，以一张特殊攻城事件卡的形式出现。

| 序号 | 城池名 | 别名 | 特色 | 出现月份 |
|------|--------|------|------|---------|
| 1 | 边陲镇 | 枯骨关 | 蛮荒边境，军力为王 | 月≥8 |
| 2 | 商贸城 | 金沙渡 | 商业重镇，需要粮草 | 月≥18 |
| 3 | 学宫城 | 墨渊阁 | 文化中心，需要谋略 | 月≥28 |
| 4 | 圣城 | 天枢殿 | 宗教/民心圣地 | 月≥38 |
| 5 | 王城 | 龙椅台 | 最终目标，全面检验 | 月≥50 |

### 2.3 四位可攻略 NPC

| NPC | 来源城池 | 身份 | 性格标签 | 影响属性轴 |
|-----|---------|------|---------|-----------|
| **阿蛮** | 第一城·枯骨关 | 蛮族女战俘 | 野性、直率、不服输 | 兵力 / 民心 |
| **红袖** | 第二城·金沙渡 | 商会大小姐 | 精明、妩媚、逐利 | 粮草 / 谋略 |
| **清照** | 第三城·墨渊阁 | 敌国才女/间谍 | 清冷、才学、国仇 | 谋略 / 民心 |
| **巫月** | 第四城·天枢殿 | 神殿祭司 | 神秘、温柔、暗藏疯狂 | 民心 / 谋略 |

## 三、数值系统

### 3.1 四维可见属性

所有属性初始值 50，范围 0-100。任意属性触及 0 或 100 即触发对应崩溃结局。

| 属性 | 代码key | 归零结局 | 满溢结局 |
|------|---------|----------|----------|
| 兵力 | military  | 兵变（士兵哗变弑主） | 军阀（将领拥兵架空你） |
| 粮草 | food  | 饥荒（全军饿散） | 腐粮（粮仓爆满引贪腐） |
| 民心 | morale  | 暴动（百姓揭竿而起） | 圣人（被神化→被联军围剿） |
| 谋略 | intel  | 中伏（情报全无被伏击） | 众叛（太多疑→幕僚离散） |

### 3.2 隐藏属性

| 属性 | 代码key | 初始值 | 说明 |
|------|---------|--------|------|
| NPC好感度 | aman_aff / hongsiu_aff / qingzhao_aff / wuyue_aff | 0（未解锁时无效） | 纳入后宫后激活，范围0-100 |
| 杀伐 | ruthless | 0 | 选择残忍选项时+1，只增不减 |
| 仁义 | righteous | 0 | 选择仁慈选项时+1，只增不减 |
| 色欲 | lust | 0 | 宠幸NPC/选择风流选项时+1 |
| 暧昧计数 | romance_count | 0 | 选择含romance:1隐藏效果的选项时+1 |
| 后宫人数 | harem_count | 0 | 每纳入一位NPC+1，最大4 |
| NPC张力 | 如 aman_v_hongsiu | 0 | 两位NPC间的冲突值 |

### 3.3 数值阻尼系统

防止属性过快触及极值导致玩家过早死亡。当属性被推向极端时，施加衰减：

```javascript
function dampen(current, change) {
  if (change === 0) return 0;
  const pushingHigh = change > 0 && current >= 75;
  const pushingLow  = change < 0 && current <= 25;
  if (!pushingHigh && !pushingLow) return change;
  let factor = 1;
  if (pushingHigh) factor = current >= 90 ? 0.5 : 0.75;
  if (pushingLow)  factor = current <= 10 ? 0.5 : 0.75;
  const dampened = Math.round(change * factor);
  if (dampened === 0) return change > 0 ? 1 : -1;
  return dampened;
}
```

- 属性 0-74 / 26-100 之间：无衰减
- 属性 75-89 / 11-25：推向极端的力度 ×0.75
- 属性 90-100 / 0-10：推向极端的力度 ×0.5
- 最小变化量保证 ±1（不会完全卡死）

## 四、卡牌系统

### 4.1 卡牌数据结构

每张卡牌是一个 JS 对象，结构如下：

```javascript
{
  id: 'card_001',           // 唯一ID
  ch: 'aman',               // 所属角色：'aman'/'hongsiu'/'qingzhao'/'wuyue'/'camp'(通配)/'war'(战争)
  tier: 'base',             // 层级：'base'(60%权重) / 'deep'(30%) / 'rare'(10%)
  theme: '沙场点兵',         // 主题分类（用于卡面显示）
  title: '逃兵',             // 卡牌标题
  t: '校场上抓到三个逃兵...', // 正文
  
  // 多周目文本（可选）
  t2: '逃兵包袱里有一封...',   // 替换文本
  t2_req: ['end_defeat_1'],   // 需解锁特定结局后才显示t2
  
  // 条件门控（可选）
  o: 1,                       // once=1 表示只出现一次
  mp: 2,                      // 最低周目数要求
  cm: 10,                     // 最低月份要求
  cs: {military:[60,100]},    // 可见属性条件（军力在60-100之间才出现）
  ca: {aman_aff:40},          // 好感度条件（阿蛮好感≥40才出现）
  ct: {aman_v_hongsiu:5},     // 张力条件
  
  // 左选项
  l: {
    t: '杀一儆百。',           // 选项文本
    f: {military:8, morale:-10}, // 可见属性变化
    h: {ruthless:1},            // 隐藏属性变化
    a: ['card_002'],            // 连锁：将card_002洗入卡池
    fl: 'deserter_killed',      // 设置全局flag
    die: 'end_xxx'              // 即死结局（可选）
  },
  
  // 右选项
  r: {
    t: '军杖三十，发回营中。',
    f: {military:-5, morale:5},
    h: {righteous:1},
  }
}
```

### 4.2 卡牌主题分类

| 主题 | 代码 | 内容范围 |
|------|------|---------|
| 沙场点兵 | battle | 军事决策、练兵、军纪、战术 |
| 幕帷之间 | intrigue | 幕僚、内政、阴谋、情报 |
| 红袖添香 | romance | 后宫互动、NPC私人事件 |
| 百姓苍生 | people | 民生、赈灾、民心、治理 |
| 攻城拔寨 | siege | 城池事件、战役决策（特殊主题） |

### 4.3 加权抽卡机制

```javascript
const TIER_WEIGHT = { base: 60, deep: 30, rare: 10 };
```

在 `drawCard()` 中，从当前可用卡池中按权重随机抽取。同时有优先机制：当有设置 flag 的情感关键卡可抽时，30% 概率强制优先抽到它。

### 4.4 连锁洗入机制（Chain System）

核心概念：某些卡牌在游戏初始化时被"雪藏"（不在卡池中），只有当玩家在另一张卡上做出特定选择后，才被洗入卡池。

实现原理：

1. `chainSet()` 函数在游戏开始前扫描所有卡牌的 `a` 数组，收集所有"被引用为后续"的卡牌 ID
2. `initPool()` 在构建初始卡池时，排除这些被引用的卡牌
3. `apply()` 在处理选择时，如果该选项有 `a` 数组，将对应卡牌 ID 推入 `G.pool`

```javascript
// 示例：主线事件连锁
// 起始卡
{id:'aman_arc_01', ..., r:{..., a:['aman_arc_02']}}
// aman_arc_02 开局被雪藏，只有选了起始卡的右选项后才被洗入
{id:'aman_arc_02', o:1, ...}
```

### 4.5 Flag 标记系统

通过 `fl` 字段在选项中设置全局标记（存储在 `G.flags` Set 中）。Flag 可用于：

1. `checkEnd()` 中判定特定结局（如情感结局需要 flag + 好感度同时满足）
2. 后续卡牌的出现条件判定
3. 攻城事件中的特殊判定

### 4.6 多周目文本替换

卡牌可选字段 `t2`（替换文本）和 `t2_req`（替换条件数组）。当玩家在之前的周目中解锁了 `t2_req` 中指定的结局后，该卡牌的文本自动替换为 `t2`，展示更深层的"真相"。

```javascript
// 渲染卡牌时的判定逻辑
function getCardText(card) {
  if (card.t2 && card.t2_req) {
    const unlocked = card.t2_req.some(eid => SD.unlocked.includes(eid));
    if (unlocked) return card.t2;
  }
  return card.t;
}
```

## 五、攻城系统

### 5.1 攻城事件卡

攻城卡是特殊的 `tier:'rare'` + `o:1` 卡牌，在特定月份后自动洗入卡池。

```javascript
{
  id: 'siege_1',
  ch: 'war',
  tier: 'rare',
  o: 1,
  cm: 8,                    // 月≥8后才出现
  theme: '攻城拔寨',
  title: '枯骨关',
  t: '斥候来报：边陲小镇守军不足五百...',
  l: {
    t: '按兵不动，继续积蓄。',
    f: {military:5, morale:5},
    h: {}
    // 不攻城，卡牌被标记used但3-5回合后重新洗入（条件提高）
  },
  r: {
    t: '出兵！',
    f: {},
    h: {},
    siege: {
      city: 1,
      req: { military: 40 },
      win: {
        f: { food: 15, morale: 10 },
        unlock_npc: 'aman',          // 解锁NPC
        next_siege: 'siege_1b'       // 如按兵不动后重来，洗入更难版本
      },
      lose: { die: 'end_defeat_1' }
    }
  }
}
```

### 5.2 攻城判定逻辑

在 `apply()` 函数中检测 `siege` 字段：

```javascript
if (ch.siege) {
  const s = ch.siege;
  let pass = true;
  for (const k in s.req) {
    let threshold = s.req[k];
    // NPC后宫buff：每位在后宫的NPC降低对应需求5点
    if (G.harem.includes('aman') && k === 'military') threshold -= 5;
    if (G.harem.includes('hongsiu') && k === 'food') threshold -= 5;
    if (G.harem.includes('qingzhao') && k === 'intel') threshold -= 5;
    if (G.harem.includes('wuyue') && k === 'morale') threshold -= 5;
    if (G[k] < threshold) pass = false;
  }
  if (pass) {
    // 攻克成功
    G.cities_taken++;
    if (s.win.f) { /* 应用奖励数值 */ }
    if (s.win.unlock_npc) { /* 触发NPC纳入事件 */ }
  } else {
    // 攻城失败 → 即死结局
    return s.lose.die;
  }
}
```

### 5.3 按兵不动的处理

选择"按兵不动"后：
- 该攻城卡从 `used` 中移除，并重新放回卡池
- 设置 `siegeDelay[city] = 当前月份 + 5`，延迟期内不可再次抽到
- 每次观望会让该城攻城需求提高 5 点，最高提高 15 点
- 成功攻下该城后，清除对应的延迟与难度记录

### 5.4 NPC 解锁流程

攻克城池后，不会自动获得 NPC。而是触发一张"NPC 纳入决策卡"：

```javascript
{
  id: 'npc_aman_offer',
  ch: 'aman',
  tier: 'rare',
  o: 1,
  theme: '红袖添香',
  title: '蛮族战俘',
  t: '在战俘中，一个浑身是伤的蛮族女子格外引人注目。她被五个士兵按着仍在挣扎，像一匹不肯被驯服的野马。',
  l: {
    t: '放她回草原。',
    f: {morale:5},
    h: {righteous:1}
  },
  r: {
    t: '纳入后宫。',
    f: {military:5, morale:-5},
    h: {lust:1, harem_count:1},
    fl: 'aman_recruited',
    // 激活阿蛮的专属卡池（10-15张）
    activate_npc: 'aman'
  }
}
```

选择"纳入后宫"后：
1. 设置 flag `aman_recruited`
2. `G.harem` 数组加入 `'aman'`
3. 阿蛮的所有专属卡牌解锁进入卡池
4. 阿蛮的好感度初始化为 30
5. 后续攻城获得阿蛮的 buff

### 7.3 checkEnd() 完整逻辑

```javascript
function checkEnd() {
  // 第一优先级：即死flag
  if (G.flags.has('retire_ready')) return 'end_retire';
  
  // 第二优先级：NPC情感结局（条件严格的先判）
  if (G.flags.has('aman_sub') && G.aman_aff >= 75 && G.military >= 50) return 'end_aman_sub';
  if (G.flags.has('aman_fall') && G.aman_aff >= 75 && G.romance_count >= 1) return 'end_aman_fall';
  if (G.flags.has('hongsiu_sub') && G.hongsiu_aff >= 75 && G.food >= 50) return 'end_hongsiu_sub';
  if (G.flags.has('hongsiu_fall') && G.hongsiu_aff >= 75 && G.romance_count >= 1) return 'end_hongsiu_fall';
  if (G.flags.has('qingzhao_sub') && G.qingzhao_aff >= 75 && G.intel >= 50) return 'end_qingzhao_sub';
  if (G.flags.has('qingzhao_fall') && G.qingzhao_aff >= 75 && G.romance_count >= 1) return 'end_qingzhao_fall';
  if (G.flags.has('wuyue_sub') && G.wuyue_aff >= 75 && G.morale >= 50) return 'end_wuyue_sub';
  if (G.flags.has('wuyue_fall') && G.wuyue_aff >= 75 && G.romance_count >= 1) return 'end_wuyue_fall';
  
  // 第三优先级：行为结局
  if (G.ruthless >= 12 && G.military >= 70) return 'end_tyrant';
  if (G.righteous >= 12 && G.morale >= 70) return 'end_saint';
  if (G.lust >= 10 && G.harem_count >= 3) return 'end_lustking';
  // 众叛亲离：至少两名已纳入NPC，且所有已纳入NPC好感<30
  const activeNPCs = G.harem || [];
  if (activeNPCs.length >= 2 && activeNPCs.every(npc => (G[npc+'_aff']||0) < 30) && G.month >= 30) return 'end_abandoned';
  
  // 第四优先级：多周目隐藏（严格的先判）
  if (SD.playthrough >= 5 && G.month >= 30 && ['aman','hongsiu','qingzhao','wuyue'].every(n => G.harem.includes(n) && (G[n+'_aff']||0) >= 60)) return 'end_harem_king';
  if (SD.playthrough >= 3 && G.cities_taken >= 5 && G.month < 60 && G.military >= 30 && G.military <= 70 && G.food >= 30 && G.food <= 70 && G.morale >= 30 && G.morale <= 70 && G.intel >= 30 && G.intel <= 70) return 'end_emperor';
  if (G.cities_taken >= 5) return 'end_unify';
  
  // 第五优先级：属性崩溃
  if (G.military <= 0) return 'end_mil_0';
  if (G.military >= 100) return 'end_mil_100';
  if (G.food <= 0) return 'end_food_0';
  if (G.food >= 100) return 'end_food_100';
  if (G.morale <= 0) return 'end_mor_0';
  if (G.morale >= 100) return 'end_mor_100';
  if (G.intel <= 0) return 'end_int_0';
  if (G.intel >= 100) return 'end_int_100';
  
  // 生存保底
  if (G.month >= 60) return 'end_warlord';
  
  return null;
}
```

## 九、存档系统

### 9.1 持久化存储

使用 `localStorage`（本地 HTML 文件运行），key 前缀 `warlord2_` 防冲突。

```javascript
const Store = {
  save(k, v) { try { localStorage.setItem('warlord2_' + k, JSON.stringify(v)); } catch(_){} },
  load(k) { try { const v = localStorage.getItem('warlord2_' + k); return v ? JSON.parse(v) : null; } catch(_){ return null; } },
  del(k) { try { localStorage.removeItem('warlord2_' + k); } catch(_){} }
};
```


**G（Game State）——当前对局状态：**

### 9.3 中途存档与恢复

每次选卡后自动存档 G 状态。关闭网页重新打开时，检测是否有未完成的对局并恢复。注意 `Set` 需要序列化为数组。

```javascript
function saveGame() {
  const sg = Object.assign({}, G);
  sg.flags = Array.from(G.flags);
  sg.used = Array.from(G.used);
  sg.curId = G.cur ? G.cur.id : null;
  delete sg.cur;
  Store.save('game', sg);
}

function loadGame() {
  const sg = Store.load('game');
  if (!sg) return false;
  Object.assign(G, sg);
  G.flags = new Set(sg.flags);
  G.used = new Set(sg.used);
  if (sg.curId) G.cur = CARDS.find(c => c.id === sg.curId);
  return true;
}
```
